Для данного проекта использовались следующие инструменты:
  IDE - PyCharm
  Язык - Python
  Framework - Pytest
  Report tool - Allure
  Паттерн проектирования - Page Object Model

В проекте были сделаны два тестовых сценария:
  позитивный сценарий - test_scenario_positive.py
  негативный сценарий - test_scenario_negative.py

В негативном сценарии я решил проверить незаполнение обязательного поле ввода 'Name'.
Тоесть по ценарию ожидаемый alert не появляется и система переводит текущий курсор на не заполненое пустое поле Name.
Для информации, поведение системы в этом случае не совсем удобно с точки зрения User Experience. Поле должно
подсвечиваться красным с сообщением типо "Заполните обязательное поле" под строкой ввода. А текущее поведение, где
курсор переходит на пустое поле может быть не сразу заметен и не понятен соответсвено.
Так же можно было протестировать поле ввода имейла на корректный формат, но в данном случае есть ошибка и
данное поле принимает любой формат, поэтому негативный сценарий тут сделать не получиться.

ВНИМАНИЕ!
В сценарии я часто использовал системные задержки time.sleep()
В web автоматизации это считается дурным тоном и не оптимально, НО я добавлял их специально
для наглядной визуализации, чтобы можно было успеть увидеть весь процесс заполнения формы.
